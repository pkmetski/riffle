# Annotations Panel — Design Spec (Issue #73)

**Date:** 2026-06-18  
**Branch:** pkmetski/surabaya  
**Issue:** [#73 — feat(annotations): per-book annotations list & navigate](https://github.com/pkmetski/riffle/issues/73)

---

## Summary

Add a slide-in annotations panel to the EPUB reader that lists all of the open book's highlights, notes, and bookmarks in reading-position order. Tapping a row navigates to that annotation; rows can be deleted (and bookmarks renamed) from the list.

---

## Scope

**In scope:**
- Per-book unified annotations list (highlights + bookmarks, ordered by reading position)
- Tap-to-navigate via Readium `Locator` (reconstructed from stored CFI)
- Delete from list (soft-delete, removes in-reader decoration)
- User-editable bookmark titles with auto-generated defaults
- Toolbar entry point (always-visible icon button next to TOC)

**Out of scope:**
- Cross-library "all my highlights" view
- Editing a highlight's note from the list (done in-reader by tapping the decoration)
- Recoloring a highlight from the list

---

All new/changed files are confined to the reader feature and the data layer. No changes to audiobook player code (`PlayerListSheet`, `AudiobookPlayerViewModel`, etc.).

**New files:**

## Data Model

### New columns on `AnnotationEntity`

| Column | Type | Default | Purpose |
|--------|------|---------|---------|
| `spine_index` | `INTEGER NOT NULL` | `0` | Sort order across chapters |
| `progression` | `REAL NOT NULL` | `0.0` | Sort order within a chapter |
| `bookmark_title` | `TEXT NOT NULL` | `""` | User-editable title (bookmarks only; ignored for highlights) |

Room migration: `MIGRATION_N_(N+1)` adds all three columns via `ALTER TABLE annotations ADD COLUMN …`. Existing rows get the defaults; the list will sort them stably.

`Annotation` domain object gains the same three fields (`spineIndex: Int`, `progression: Double`, `bookmarkTitle: String`).

### AnnotationStore additions

```
fun renameBookmark(id: String, title: String, now: Long)
```

`createBookmark` gains a `bookmarkTitle: String` parameter. The title is generated by the ViewModel (`toggleBookmark()` has access to both the Readium locator and `tocEntries`), then passed in. The store stores it verbatim.

## EpubBookmarkTitleBuilder

Generates the default bookmark title from the Readium locator and TOC at creation time.

**Logic (in priority order):**
1. If a TOC entry matches `locator.href` and has a non-blank title:  
   `"<chapter title> · <within-chapter progression as %>"` e.g. `"The Egg · 34%"`
2. If `locator.locations.totalProgression` is non-null:  
   `"<whole-book totalProgression as %>"` e.g. `"34%"`
3. Final fallback (totalProgression not yet computed by Readium):  
   `"<within-chapter progression as %>"` e.g. `"34%"`

All progression values come from `locator.locations` at toggle time; percentage is `(value * 100).roundToInt()`. The builder is a pure function — inputs are the locator and the TOC entry list, output is a `String`.

## Bookmark Creation Flow

The corner ribbon toggle behaviour is **unchanged**: tap to add, tap again to remove. No creation dialog. The title is auto-generated silently by `EpubBookmarkTitleBuilder` when the bookmark is created. Rename is available from the annotations panel overflow menu.

## Annotations Panel

### Entry point

A new `IconButton` in the reader `TopAppBar`, between the TOC and Format buttons. Icon: `Icons.Filled.Bookmarks`. Visible whenever `state is ReaderState.Ready` (always, regardless of annotation count). Opening the annotations panel closes the TOC if open, and vice versa.

### Surface

`AnnotationsPanel` is a slide-in side panel following `TocPanel`'s pattern exactly — same entry/exit animation, same backdrop, same dismissal on backdrop tap or back press. It is not a `ModalBottomSheet`. No code is shared with `TocPanel`; both are independent composables.

### Content

`LazyColumn` of all non-deleted annotations for the open book, sorted `spine_index ASC, progression ASC`. The sort order is computed by the `AnnotationDao` query, not in the ViewModel.

**Panel header:** "Annotations"

**Empty state:** "No annotations yet." — `bodyMedium`, `onSurfaceVariant`

### Row design

Both row types follow the same layout: 32 dp lead box | text column (title + optional subtitle, `weight(1f)`) | trailing `MoreVert` button.

**Highlight row**
- Lead: filled circle (16 dp) in the highlight's `HighlightColor`
- Title: `textSnippet`, up to 2 lines, ellipsized — `bodyLarge`, `onSurface`
- Subtitle: first 60 chars of `note` if non-null — `labelMedium`, `onSurfaceVariant`
- Trailing overflow: **Delete** only

**Bookmark row**
- Lead: `Icons.Filled.Bookmark`, `onSurfaceVariant`
- Title: `bookmarkTitle` (auto-generated or user-renamed), up to 2 lines — `bodyLarge`, `onSurface`
- Subtitle: none
- Trailing overflow: **Rename** + **Delete** (matching audiobook player's bookmark overflow)

**Rename dialog** for bookmarks matches the audiobook player's `BookmarkCreateDialog` pattern: pre-filled `OutlinedTextField` with the current title; no position label or chip suggestions needed (EPUB rename is simpler than audiobook).

## ViewModel Additions

| Addition | Type | Description |
|----------|------|-------------|
| `annotationsPanelVisible` | `StateFlow<Boolean>` | Drives panel open/close |
| `annotations` | `StateFlow<List<Annotation>>` | Live, position-ordered, non-deleted |
| `openAnnotationsPanel()` | `fun` | Sets visible = true |
| `closeAnnotationsPanel()` | `fun` | Sets visible = false |
| `navigateToAnnotation(id: String)` | `fun` | Reconstructs `Locator` via `cfiStringToLocator(annotation.cfi)`, emits on `annotationNavigationEvents` |
| `renameBookmark(id: String, title: String)` | `fun` | Delegates to `AnnotationStore.renameBookmark()` |
| `annotationNavigationEvents` | `Flow<Locator>` | Consumed by the screen, same pattern as `searchNavigationEvents` |

The CFI → Locator reconstruction reuses the existing `cfiStringToLocator()` helper (EpubReaderViewModel line ~980).

## Tests

Following the issue's acceptance criteria, JVM unit tests cover:

1. **List contents and ordering** — highlights + bookmarks sorted by `(spineIndex ASC, progression ASC)`; tombstoned annotations absent
2. **Navigate-to-Locator** — `navigateToAnnotation(id)` reconstructs the correct `Locator` from the stored CFI and emits it on `annotationNavigationEvents`
3. **Rename** — `renameBookmark(id, title)` updates `bookmarkTitle`; the live `annotations` list reflects the change
4. **Delete from list** — soft-deletes the annotation, removes it from the live `annotations` list; in-reader decorations are driven by the same store so they disappear too
5. **`EpubBookmarkTitleBuilder`** — chapter-title path, fallback path, blank-chapter-title fallback

Migration test follows the `MigrationTest.kt` pattern: insert rows at version N, run migration, assert new columns have correct defaults, pre-existing data preserved.
